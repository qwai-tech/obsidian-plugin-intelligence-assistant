# PKM Principles

My core principles for building an effective second brain.

> **Meta-principle:** A note is only as valuable as the frequency with which you return to it. Build for retrieval, not archiving.

## Capture Everything, Filter Later

Don't judge ideas at capture time. The cost of a missed insight far exceeds the cost of storing a mediocre one. Use quick-capture to reduce friction to near zero.

- Minimum viable note: title + one sentence
- Mobile capture → inbox → process weekly
- Fleeting notes are ephemeral; permanent notes are atomic

## Progressive Summarization

Layer notes with increasing compression:

1. **Raw capture** — full source text or transcript
2. **Highlighted passages** — pull out key ideas (bold)
3. **Bold highlights** — the single most important idea per passage
4. **Executive summary** — 2–3 sentences at the top of every note

*Source: Building a Second Brain, Tiago Forte (ch. 7)*

## PARA Method

Organise by actionability, not topic:

| Folder | Contains | Lifespan |
|--------|----------|----------|
| **Projects** | Active goals with deadlines | Weeks–months |
| **Areas** | Ongoing responsibilities | Indefinite |
| **Resources** | Reference material | Indefinite |
| **Archive** | Completed/inactive items | Permanent |

Key insight: move notes *toward* the project that needs them most.

## Link Generously

Every note should link to at least 2 others. Orphan notes are forgotten notes.

- Prefer **concept links** over tag links (`[[Spaced Repetition]]` not `#memory`)
- Write a "related notes" section when linking is non-obvious
- The graph view should resemble a web, not a tree

Related: [[LLM Architecture]], [[Reading List]]

## Intermediate Packets

Break large projects into small, reusable knowledge objects:

- A summary of a book chapter > the whole book
- A distilled principle > the raw research
- A worked example > a definition

Packets should be self-contained — readable without context.

## Retrieval over Archiving

Titles should be **answers**, not topics:
- ✗ "Attention Mechanism"
- ✓ "How transformers avoid recomputing KV pairs"

Use the search-engine heuristic: would you type this title into a search bar when you *need* this information?
