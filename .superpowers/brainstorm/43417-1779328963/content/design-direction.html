Chat UI — 整体设计方向

当前界面存在几个问题：头部控制栏视觉过重、禁用按钮占用过多空间、嵌套卡片感太强。以下是三种重设计方向，选择最符合你期望风格的。

A · 极简主义

移除所有卡片边框，用留白区分层次。头部一行内展示所有控制，输入框完全扁平化。禁用功能仅以图标形式存在，不显示文字标签。参考：Linear、Notion、Claude.ai

B · 侧边栏配置

将模式/模型/RAG/网搜等控制移到左侧固定面板，聊天区域独占右侧全部空间。侧边栏可折叠。像 VS Code 的 Activity Bar 一样，配置与对话完全分离。

C · 精炼现有

保留现有整体结构，针对性优化：头部压缩为单行、移除多余边框、输入框功能区紧凑化、禁用项改为纯图标+tooltip（无文字标签）、启用项以高亮区分。改动量最小，风险最低。

💡 提示：点击卡片选择，也可以在终端里描述你的想法或者说"混合A和C"之类的。
