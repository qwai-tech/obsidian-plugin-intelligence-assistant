# i18n 多语言支持设计文档

**日期：** 2026-05-20
**状态：** 已批准

---

## 概述

为 Intelligence Assistant 插件添加国际化支持，初始支持英文（en）和简体中文（zh），语言随 Obsidian 界面语言自动切换。

---

## 目标

- 所有 UI 字符串（设置页、聊天界面、弹窗、Notice）支持多语言
- 语言跟随 Obsidian 的 moment.js locale 设置，无需用户在插件内单独配置
- 翻译文件使用 JSON 格式，方便社区贡献者参与翻译
- 零新增 bundle 体积压力（两个 locale JSON 静态打包进 main.js）

---

## 技术选型

使用 **i18next**（[npmjs.com/package/i18next](https://www.npmjs.com/package/i18next)）。

选择理由：
- 业界标准，API 稳定
- 支持插值（`{{variable}}`）、fallback 语言、嵌套 key
- 同步初始化模式，适合 Obsidian 插件的单线程环境
- 无需 i18next-browser-languagedetector 等插件，直接读取 Obsidian 暴露的 `window.moment.locale()`

---

## 文件结构

```
src/i18n/
  index.ts          # 初始化 i18next，导出 t()
  locales/
    en.json         # 英文翻译（基准，必须完整）
    zh.json         # 简体中文翻译
```

---

## 模块接口

`src/i18n/index.ts` 对外只暴露两个符号：

```typescript
export function initI18n(): void;
export const t: TFunction;
```

调用方：
```typescript
import { t } from '@/i18n';
// 简单 key
t('settings.tabs.general')              // → "General" / "通用"
// 带插值
t('notices.error', { message: '...' }) // → "Error: ..." / "错误：..."
```

## 初始化

在 `main.ts` 的 `onload()` 入口处、任何 UI 渲染前调用一次：

```typescript
import { initI18n } from './src/i18n';

async onload() {
  initI18n();
  // ... 其余初始化
}
```

`initI18n` 内部逻辑：

```typescript
import i18next from 'i18next';
import en from './locales/en.json';
import zh from './locales/zh.json';

export function initI18n() {
  const raw = window.moment?.locale() ?? 'en';
  // 将 'zh-cn', 'zh-tw', 'zh-hk' 等归一为 'zh'
  const lang = raw.startsWith('zh') ? 'zh' : 'en';

  i18next.init({
    lng: lang,
    fallbackLng: 'en',
    resources: {
      en: { translation: en },
      zh: { translation: zh },
    },
    interpolation: { escapeValue: false },
  });
}

export const t = i18next.t.bind(i18next);
```

## 字符串迁移计划

迁移范围：`src/presentation/` 下所有非测试、非备份 `.ts` 文件，约 459 处字符串字面量。

迁移优先级：

| 优先级 | 文件/目录 | 大约数量 |
|--------|-----------|---------|
| 1 | `components/settings-tab.ts` + `tabs/*.ts` | ~320 |
| 2 | `components/chat/*.ts` + `components/chat/handlers/` | ~80 |
| 3 | `components/modals/*.ts` | ~40 |
| 4 | `main.ts`（Notice 消息） | ~20 |

每个文件迁移模式：
```typescript
// 文件顶部添加
import { t } from '@/i18n';

// 替换硬编码字符串
.setName(t('settings.general.pluginVersion.name'))
.setDesc(t('settings.general.pluginVersion.desc'))
```

## 错误处理

- i18next 内置 `fallbackLng: 'en'`：任何缺失 key 自动回退英文，不会出现空字符串
- 初始化不抛出异常（i18next.init 同步调用时无 Promise rejection）
- 将来新增语言：新增 `src/i18n/locales/xx.json` + 在 `resources` 中注册，无需改动其他代码
