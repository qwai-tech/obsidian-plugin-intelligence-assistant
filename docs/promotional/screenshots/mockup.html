RAG Architecture

Retrieval-Augmented Generation (RAG) combines a retrieval system with a generative model. The retrieval component searches a document store for relevant context, which is then injected into the model's prompt.


Dense retrieval uses embedding similarity (cosine/dot-product) to find semantically related chunks. Sparse retrieval uses keyword matching via BM25. Hybrid approaches combine both for best recall and precision.


Key parameters: chunk size, overlap, top-k, similarity threshold.
